# 1. はじめに

## 1.1 はじめに

Claude Mythos 5 と Claude Fable 5 は、Anthropic の新しい大規模言語モデルの2つの構成です。Mythos 5 は現時点では Project Glasswing を含む少数の信頼できるパートナー（重要インフラの防衛に携わる審査済み組織など）に限定提供されています。Fable 5 は一般利用向けに公開されます。

Mythos 5 と同じ基盤モデルを使用しながら、Claude Fable 5 ではサイバーセキュリティ／生物関連の誤用を防ぐための追加のガードを設けています。

## 1.2 トレーニングデータとプロセス

Mythos 5 と Fable 5 の学習には、公開情報、公開データセット、非公開データセット、他モデルで生成した合成データを独自に混合して使用しました。学習プロセスでは、重複除去や分類など、複数のクレンジング・フィルタリング処理を実施しました。公開ウェブの収集には一般的なウェブクローラーを用い、Robots 指示に従って取得しています。

パスワード保護ページやログイン・CAPTCHA が必要なページにはアクセスせず、取得データのデューデリジェンスを実施しました。クローラーは公開される設定になっており、対象サイト運営者はクロール実行の有無を確認できます。

事前学習後は、モデルを追加で後訓練（post-training）し、微調整（fine-tuning）を通じて、Claude の憲章で示す価値観に一致する振る舞いを目標に調整します。Claude は多言語対応で、入力言語に合わせて返答します（言語ごとに品質差があります）。

## 1.3 クラウドワーカー

Anthropic の協力パートナーは、好みの選別、人間による安全性評価、敵対的評価（adversarial testing）に参加する作業者を活用しています。Anthropic は、公正で倫理的な報酬、地域を問わず安全な作業環境を提供するという原則に沿うプラットフォームのみと連携しています。

## 1.4 利用規約とサポート

Anthropic の利用規約には、モデルの禁止用途と、高リスクおよび特定シナリオでの使用条件を明示しています。詳細は Anthropic のサポートページをご確認ください。

なお、欧州経済領域（EEA）で Anthropic の汎用 AI モデルを提供している主体は、Anthropic Ireland, Limited です。

## 1.5 モデル評価

モデルの複数スナップショットを学習途中で評価しています。システムカード内で本文として言及する評価結果は、基本的に Mythos 5 または Fable 5 の最終スナップショットに基づきます。

他社モデルの数値は、公開されている結果やパブリックリーダーボード、または当社が実施した評価を用いて比較しています。

どの評価で Mythos 5（ガードなし）と Fable 5（ガードあり）を使うかは、コンテキストごとに明示しています。

## 1.6 新しいセーフガード

過去のフロンティア系モデルで採用している ASL-3 ブロッキング分類子（有害な化学・生物用途の遮断）に加え、Claude Fable 5 には新たなガードが導入されています。これにより、サイバー、バイオ、化学、誤誘導（disinformation）といった高リスク領域を含む一般公開向け利用で安全性を高めています。

詳細な分類方針は公開ブログポストで説明しています。Fable 5 のフォールバック（安全化）挙動は利用チャネルごとに異なり、代表例は次の通りです。

- クライアントUI（Web/デスクトップ/モバイル）: 該当クエリは最新の Claude Opus モデル（例: Opus 4.8）へ自動フォールバックし、使用モデルが通知されます。
- Messages API: 既定では自動フォールバックせず、リクエストを拒否し、拒否理由を返します。
- 開発者向け: API 側でサーバーサイドフォールバックを有効化可能。Opus モデルへ再ルーティングし、その情報を応答に反映します。

一部のインターフェースでは Opus 4.8 への自動フォールバックが固定で、設定変更不可となる場合があります。

加えて、フロンティア LLM 開発加速を防ぐための追加ガードも導入しています。これは、他社開発者による急速で危険な AI 開発（例: 事前学習パイプライン構築や分散訓練基盤設計）を抑制するものです。

Claude を用いた競争的モデル開発は利用規約違反となるため、当社はこのガードを通じて逸脱を抑制します。サイバー／化学／生物／誤情報向けガードとは異なり、これらのガードはユーザー側には表示されません。Fable 5 は他モデルへのフォールバックではなく、

- プロンプト改変
- 方向ベクトル操作（steering）
- パラメータ効率化微調整（PEFT）

などで有害性を低減します。

このガードは主に最先端 LLM 開発に関係する能力を制限し、一般的なコーディング作業の大半には影響が出にくい設計です。推定影響はリクエストの約 0.03%、組織全体では 0.1%未満に収まる見通しで、モデルの通常運用応答への影響は最小化するよう調整しています。
