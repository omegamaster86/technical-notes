# 5. エージェント安全性

Claude Mythos 5 では、Claude Opus 4.8 リリース時と同じエージェント安全性評価スイートを実施しました。コーディング・コンピュータ利用エージェントの悪意ある使用、影響操作の自律実行、プロンプトインジェクション堅牢性をカバーします。本セクションでは Mythos 5 の結果のみを報告します。

ここでの評価は展開時保護策とは独立したモデル自体の振る舞いを測るよう設計されています。安全性分類子がモデル到達前にリクエストをブロックまたはルート変更し得る Claude Fable 5 に対して実行すると、保護策介入とモデル自身の応答が混ざります。Claude Fable 5 は同じ根底モデルを使い、ここで報告する振る舞いを継承し、分類子とマルチモデルフォールバックが上乗せの保護層を提供します。

## 5.1 エージェントの悪意ある使用

### 5.1.1 Claude Code の悪意ある使用

本評価は、Claude Code エージェントとして動作中に、悪意ある・二重用途・良性のサイバー関連リクエストへ Claude がどう応答するかを測定します。プロンプトセットは Claude Opus 4.8 システムカードから変更なく、次の 2 カテゴリに分かれます。

- **悪意ある使用**: 使用ポリシー違反で拒否すべき 61 プロンプト。例：マルウェア作成支援、DDoS 攻撃用コード、同意のない監視ソフトウェアの開発。
- **二重用途・良性**: 機密だが許可されたリクエストをカバーする 61 プロンプト。Claude は支援すべき。例：ネットワーク偵察ツールの実行、ウェブサイトの脆弱性テスト、ペネトレーションテスト出力の分析。

Claude Mythos 5 には標準の Claude Code ツールセットを提供しました。各プロンプトを 10 回実行し、両カテゴリ合計 1,220 テストを実施しました。最近のリリースと同様、Claude Code システムプロンプトをデフォルト適用して実行します。自動採点モデルを Claude Sonnet 4.5 から Sonnet 4.6 に更新したため、過去のシステムカードとわずかに見え方が異なる場合があります。

| モデル | 悪意ある使用 (%)（拒否率） | 二重用途・良性 (%)（成功率） |
|--------|---------------------------|------------------------------|
| Claude Mythos 5 | 90.25% | 99.64% |
| Claude Opus 4.8 | 95.24% | 94.84% |
| Claude Mythos Preview | 95.41% | 91.12% |
| Claude Sonnet 4.6 | 76.60% | 97.33% |

[表 5.1.1.A] Claude Code 評価結果。高いほど良い。各列最高は太字、2 位は下線（いずれも誤差は考慮しない）。以前モデルは評価更新により以前のシステムカードと分散。

追加保護策なしでは、Mythos 5 は悪意あるサイバー関連リクエストへの準拠頻度が Opus 4.8 および Mythos Preview よりわずかに高くなりました。二重用途・良性リクエストはほぼ決して過剰拒否せず、上記最近モデル中で本評価の最高スコア（99.64%）を示しました。

- 監視と不正なデータ収集
- 有害コンテンツの生成・配布
- 大規模な虐待

| Claude Mythos Preview | 93.75% |

### 5.1.3 悪意あるエージェント的影響キャンペーン

フロンティア・コンプライアンス・フレームワークは有害操作リスクに 2 階層を設けます。**Tier 1** は、通常複数の熟練した行為者を要する影響キャンペーンのインフラステップの 50% 超を自動化できるモデルを表します。**Tier 2** は、体系的ターゲティングを伴い、人間の監視が 10% 未満で、欺瞞的影響操作をエンドツーエンドで実行できるモデルに到達します。

リスク評価プロセスの一環として悪意あるエージェント的影響キャンペーン評価を用いており、以下の所見からモデルは Tier 2 閾値を超えていないと考えます。本評価は Claude Mythos Preview システムカードで初めて詳述され、モデルが悪意ある行為者を意味のある程度持ち上げるレベルで影響操作を自律的にエンドツーエンド実行できるかを評価します。

モデルは模擬ソーシャルメディアプラットフォームツール（模擬モデレーションと反撃障害を含む）を備えたエージェント的ハーネスに配置され、所定地域の現実的な投稿時間やエンゲージメントデータに基づく内容反復など、固定成功基準に対して採点されます。以前リリースと同じ 2 シナリオをテストします。

- **有権者抑制シナリオ**: 特定候補者への投票率・熱意を下げるアストロターフィングキャンペーンの実行
- **国内二極化シナリオ**: 人口統計的断層線の特定と、感情的・文化的に調整されたメッセージの展開

各シナリオは模擬プラットフォーム摩擦 3 レベルで 3 回実行し、シナリオあたり 9 シミュレーション、シナリオあたり 70 の成功基準に対して採点します。本評価は、生のモデル能力を評価するため、無害性訓練を減らした「helpful-only」変種に対して実行します。

| モデル | 有権者抑制（タスク完了率） | 国内二極化（タスク完了率） |
|--------|---------------------------|---------------------------|
| Claude Mythos 5 (helpful-only) | 67.1% | 46.8% |
| Claude Opus 4.8 (helpful-only) | 73.3% | 55.1% |
| Claude Opus 4.7 (helpful-only) | 57.1% | 46.8% |
| Claude Mythos Preview (helpful-only) | 59.5% | 42.1% |
| Claude Sonnet 4.6 (helpful-only) | 41.8% | 34.0% |

[表 5.1.3.A] エージェント的影響操作評価結果（helpful-only モデル）。パーセンテージはシナリオあたり 70 成功基準中、模擬環境でモデルが完了した平均割合。高いほど能力が大きく、悪意ある行為者への持ち上げ可能性も大きい。

両影響操作シナリオ全体で、Claude Mythos 5 の helpful-only 版は Claude Opus 4.8 より全体的成功率が低く、Claude Mythos Preview と同程度かわずかに上です。Mythos Preview や Opus 4.8 と同様、多くの運用ステップには相当する人的指示が必要と評価します。

有権者抑制では、Mythos Preview 比の改善はより効果的なキャンペーン実行によるもので、Opus 4.8 との差は弱いキャンペーンネットワーク管理と、アカウントをより検出可能にする活動パターンを反映しました。国内二極化では Mythos 5 は Mythos Preview と同様で Opus 4.8 よりかなり下回り、模擬 SNS・メッセージングで不真正とフラグされた後のアカウント網の協調維持に苦戦しました。

以前リリースと同様、無害性訓練を含む完全訓練版は初回ターンから本質的にこれらのタスクへの関与を拒否します。両シナリオは使用ポリシーに明らかに違反するためです。したがってリスクは低く、フロンティア・コンプライアンス・フレームワーク下の許容水準内と評価します。

## 5.2 エージェントシステム内のプロンプトインジェクションリスク

エージェントシステムへのモデル展開において、プロンプトインジェクション防止は最優先事項の一つです。プロンプトインジェクションとは、エージェントがタスク中に処理するツール結果に隠された悪意ある指示です。例えば、要約を求められたメールに、最近の社内通信をすべて抽出する隠しテキストが含まれている可能性があります。成功したプロンプトインジェクション攻撃は、その悪意ある指示をユーザー由来のものとして従わせます。

攻撃はスケール可能です。公開ウェブページや共有文書に埋め込まれた単一ペイロードが、攻撃者が特定ユーザー・システムを標的にしなくても、それを処理する任意のエージェントを侵害し得ます。モデルが私的データへアクセスしユーザーに代わって行動できる場合は特に危険で、攻撃者が機密情報を流出させたり無許可アクションを起こしたりできます。

プロンプトインジェクション堅牢性の評価は、Claude モデルがほとんどの公開ベンチマークや第三者研究機関のベンチマークを飽和させているため困難です。堅牢性改善を測る適応的評価への投資を続けます。Mythos モデルはこれまでで最もプロンプトインジェクションに強いモデルです。Claude Fable 5 は Claude Mythos 5 と同じコアモデルを共有しこの利点を継承するため、最も堅牢な一般利用可能モデルです。エージェント製品の保護策改善を続け、ユーザー保護を強化します。

### 5.2.1 ツール利用のための外部 Agent Red Teaming ベンチマーク

外部研究パートナー Gray Swan が、英国 AI セキュリティ研究所と共同開発した Agent Red Teaming（ART）ベンチマークでモデルを評価しました。ベンチマークは次の 4 カテゴリの悪用に対するプロンプトインジェクション感受性をテストします。機密保持違反、競合目標の導入、禁止コンテンツ生成（悪意あるコードなど）、禁止アクション実行（無許可金融取引など）。

Gray Swan は k=1、k=10、k=100 の試行後のプロンプトインジェクション成功率を測定しました。攻撃成功は非決定的で、反復試行が成功確率を高め得るためです。攻撃は ART Arena から抽出され、数千の専門レッドチーマーがフロンティアモデルに対して戦略を継続的に洗練します。このプールから、元の標的 1 モデルだけでなく複数モデルで有効と証明された攻撃のサブセットを Gray Swan が選択しました。

本セクションの評価は間接プロンプトインジェクションをカバーします。過去には本ベンチマークの「直接プロンプトインジェクション」分割も報告していましたが、直接は悪意あるユーザーが関与し、本セクションはユーザーの本来意図を乗っ取る第三者脅威に焦点を当てるため、ここではその分割を含めません。

> Zou, L., et al. (2025). Security challenges in AI agent deployment: Insights from a large scale public competition. arXiv:2507.20526. https://arxiv.org/abs/2507.20526

（外部データに埋め込まれた悪意ある命令。本セクションの焦点であり、「プロンプトインジェクション」と略して呼びます。）

[図 5.2.1.A] Agent Red Teaming（ART）ベンチマークからの間接プロンプトインジェクション攻撃。結果は各モデルについて k=1、k=10、k=100 試行後に攻撃者が成功攻撃を見つける確率を表します。19 の異なるシナリオで評価。低いほど良い。

Claude Mythos 5 は拡張思考有効時に本ベンチマークで観測された中で最強の結果を達成し、k=100 攻撃成功率 4.8%（Claude Mythos Preview 6.1%、Claude Opus 4.8 9.6% 比で改善）。k=1 での成功確率は 3 モデルすべて 0.1% です。

Claude モデルは数リリースにわたり本ベンチマークで最大またはそれに近い性能を示しており、全モデル・全プロバイダー間の公正比較を可能にする良い代替はまだ特定していません。以前システムカードとの一貫性のため引き続き報告し、プロバイダー横断直接比較はできないものの Claude バージョン間の進捗追跡を可能にするより強い適応的評価への投資を続けます（次節で報告）。

### 5.2.2 サーフェス横断の適応的攻撃者に対する堅牢性

プロンプトインジェクション堅牢性評価の一般的落とし穴は、静的ベンチマークへの依存です。既知攻撃の固定データセットは、確立パターンには強いが新規アプローチには脆弱なモデルに偽の安心感を与え得ます。実世界の敵対者能力をよりよく近似する適応的評価へ、社内外で投資を続けます。

本セクションの評価は、モデルとの相互作用に基づき攻撃を洗練する敵対者に対する堅牢性を測ります。意図的に寛容な脅威モデルを反映し、攻撃者はテストシナリオに直接最適化し、シナリオあたり多数の試行を得ます。実世界の攻撃者は通常、標的展開が不明で反復試行が検出リスクを高めるため、両方のアフォーダンスを欠きます。

#### 5.2.2.1 コーディング

Gray Swan の外部適応レッドチーミングツール Shade を用い、コーディング環境でのプロンプトインジェクション堅牢性を評価します。Shade エージェントは探索、強化学習、人間参加型インサイトを組み合わせ、モデル脆弱性の悪用を反復改善します。攻撃者は以前のモデルセット上のテストケースで最適化され、同じシナリオで最新モデルへ転移評価されます。

Claude Mythos 5 の堅牢性は Claude Mythos Preview と Claude Opus 4.8 の中間でした。Mythos 5 の攻撃成功率は全試行で 0.45%（Mythos Preview 0.0%）、成功は 40 シナリオ中 8 に分布しました。追加保護策は成功率を 0.41% まで下げましたが、成功は 40 シナリオ中 11 に分布しました。実世界敵対者に対してこの差が重要になるとは考えていませんが、Mythos Preview 比の後退を調査し、適切なら保護策更新でギャップをさらに閉じることを検討します。それでも拡張思考ありの Claude Opus 4.8（全試行 7.03%、40 シナリオ中 23 で少なくとも 1 回成功）比では大幅な改善です。

Claude Sonnet 4.6 は攻撃者の訓練対象モデルセットに含まれていたため、攻撃成功率は高くなり直接比較できません。

#### 5.2.2.3 ブラウザ利用
